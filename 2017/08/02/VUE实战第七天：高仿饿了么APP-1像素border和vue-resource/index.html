Vue2.0的vue-resource

  • Vue1.x推荐使用的是ajax插件vue-resource,但是从Vue更新到2.x,主流的Vue项目都选择axios来完成ajax请求,而大型项目都会使用Vuex来管理数据

  1. 使用cnpm安装axios

  2. 引用插件

  • 安装其他插件的时候,可以直接在main.js中引入并Vue.use(),但是axios不能use,只能每个需要发送请求的组件中及时引入插件。两种开发思路:一:在引入axios之后,修改原型链;二是结合Vuex,封装一个aciton。具体操作如下:
  • 方案一:改写原型链

  1. 首先在main.js中引入axios

  2. 此时如果在其他的组件中,是无法使用axios命令的,单如果将axios改写为Vue的原型属性,就能解决这个问题

  3. 在main.js中添加了这两行代码之后,就能直接在组件的methods中使用$ajax命令

  • 方案二:在Vuex中封装
  1. ①mutations: 从结果上看,mutations类似于事件,用于提交Vuex中的状态state。有一个固有参数state,接收的是Vuex中的state对象
  2. ②action: 类似于mutations,主要区别在于,ation可以包含一部操作,而且可以通过action来提交mutations。有一个固有的参数context,但是context是state的父级,包含state，getters

  3. Vuex的仓库是store.js,将axios引入,并在action添加新的方法

  4. 即使在main.js中引入了axios,并改写了原型链,也无法在store.js中直接使用$ajax命令；所以这两种方案是相互独立的。在组件发送请求的时候,需要使用this.$store.dispatch来分发

  5. submitForm是绑定在组件上的一个方法,将处罚saveForm,从而通过axios向服务器发送请求

  4. 构建核心仓库store.js

    • Vuex应用的状态state都应当存放在store.js里面,Vue组件可以从store.js里面获取状态,可以把store通俗的理解为一个全局变量的仓库。但是和单纯的全局变量又有区别,主要体现在当store中的状态发生改变时,相应的vue组件也会得到高效更新。
